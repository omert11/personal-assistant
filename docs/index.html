ask-first
Önce Sor

Kullanıcıya yöneltilen her soru, seçim veya onay düz metinle değil, yalnızca AskUserQuestion tool çağrısıyla sorulur. Düz metin soru tamamen yasaktır.

Kural mutlaktır: tek bir evet/hayır onayı bile ("devam edeyim mi?" gibi tek satır metin yasak — yerine ["Evet","Hayır"] tool çağrısı); commit/destructive/geri-alınamaz onaylarda metinle "onaylıyor musun?" yasak; çoklu seçenekte markdown 1. 2. 3. liste yasak. Felsefe: varsayım = hata, sormak = kalite; çelişki görünce sor, bilgi eksikse tahmin etme.

Self-Check

Metne ? yazmadan önce dur: (1) kullanıcıya soru mu? → AskUserQuestion; (2) "A mı B mi?" fragment yasak; (3) "onaylıyor musun" formları hep tool. Önerilen seçenek ilk sırada (Recommended) etiketiyle.

workflow
Görev Takibi + Goal + Subagent + Background

Çok adımlı işte görevler TaskCreate/TaskUpdate ile canlı takip edilir; subagent/workflow task zorluğuna göre serbest — ama iş delegasyondan önce kesinleştirilir ve çıktı doğrulanır.

Görev takibi: 3+ adım / çok dosya / planlı akış başlayınca TaskCreate ile görev listesi açılır ve süreç boyunca canlı tutulur — kullanıcı akışın nerede olduğunu görsün, hiçbir adım unutulmasın.

Subagent/workflow — üç keskin şart: (1) delegasyondan önce iş kesinleştirilir — girdi, kapsam sınırı, beklenen çıktı formatı ve "ne yapılmayacak" prompt'a net yazılır, muğlak tanımla agent çağırmak yasak; (2) gereksiz kontrol önlenir — çıktıyı ana agent doğrulayacaksa subagent'a "review/test yapma, kontrol bende" sınırı konur; (3) çıktıya "doğru değilmiş" gözüyle bakılır — iddialar kendin teyit edilir, parçalar çelişki ayıklanarak birleştirilir, doğrulanamayan şey doğrulanmış gibi raporlanmaz.

Çalıştırma: bloklamayan işler varken bloklayan komut background'a alınır (subagent, uzun Bash, workflow); açık kalması gerekmeyen her Bash komutuna timeout konur.

token-efficiency
Model Ekonomisi

Subagent ve Workflow ajanlarında model her zaman bilinçli seçilir — en küçük yeterli model.

Agent frontmatter'ına her zaman açık model: yazılır (haiku/sonnet/opus/fable, tam ID veya inherit); workflow agent() çağrılarında opts.model açıkça belirtilir. Mutlak kural — sub/agent fable YASAK: hiçbir workflow agent() çağrısı veya subagent fable ile çalıştırılmaz; fable oturumunda opts.model boş bırakmak fable'ı devralır, bu yüzden her çağrıya açık model yazılır (en yükseği opus). Hazır/named workflow'lar (Workflow({name: ...})) model override taşımadığından fable oturumunda doğrudan launch edilmez — script kopyasına her agent() için açık model yazılıp scriptPath ile çalıştırılır (mekanik/finder sonnet, derin verify/judge opus); kopya mümkün değilse launch edilmez, kullanıcıya sorulur. Code review — tek kural: code-review'i tetikleyen her yer (commit, issue-workflow, ad-hoc) aynı akışı izler — fable oturumunda code-review tek general-purpose subagent'la model: opus verilip medium akışı inline koşulur (yeni Workflow spawn edilmez), fable değilse Claude kendisi inline koşar; effort tavanı medium (agent low/medium seçer, high+ yalnızca kullanıcı isterse).

Örnek
agent(prompt, { label: 'translate:ar', model: 'sonnet', schema })

commit

Kod değişikliklerini commit etmeden önce kapsamlı kalite kapısı kurar. Tüm analizi (code-review, test, rules uyumu, Plane issue, Obsidian) sessizce yapar, bulguları biriktirir, tek toplu soru bloğunda sunar, onay sonrası seçilen teslimatı (commit/push/PR/merge) uygular.

Tetikleyici

"commit", "push et", "PR oluştur", "branch aç", /commit. before-commit kuralı gereği manuel git commit yerine zorunludur.

Akış

status → branch tespit → code review (zorunlu) → test → rules tara → Plane issue eşleştir → Obsidian → tek soru bloğu → dosyaları tek tek stage + Co-Authored-By commit → teslimat. Review, built-in /code-review skill'iyle çalıştırılır; effort değişikliğin karmaşıklığına göre low/medium arasında seçilir (agent tavanı medium; high+ yalnızca kullanıcı isterse). Fable Model Barajı: ana oturum modeli Fable ise code-review tek general-purpose subagent'la model: opus verilip medium akışı inline koşulur (yeni Workflow spawn edilmez); Fable değilse Claude kendisi inline koşar (review yine yapılır, yalnızca fable devri ve kontrolsüz agent fan-out engellenir).

Öne çıkan

Code review atlanamaz — "hızlıca commit" dense bile çalışır. Effort işin karmaşıklığına göre kalibre edilir ama agent tavanı medium: tek-birkaç dosyada dar kapsamlı değişiklik (küçük mantık/test/docs/config dahil) low (küçük diff varsayılanı), birden çok dosyaya yayılan orta ölçekli iş medium. Karmaşık/güvenlik-kritik/data-mutating kod olsa bile agent medium'da kalır; high/xhigh/max yalnızca kullanıcı açıkça isterse — agent kendiliğinden yükseltmez. Bulgular küçük diye filtrelenemez/yumuşatılamaz; uncertain bulgular bile listelenir, false-positive kararını kullanıcı verir. Asla --amend, asla force push, asla git add -A.

vault + CLI · proje belleği

Nedir: Markdown tabanlı, yerel bir bilgi tabanı uygulaması; notları [[wikilink]] ile birbirine bağlayıp graf oluşturur. Resmi CLI (obsidian komutu) ile programatik okuma/yazma/arama sağlar.

Ne işe yarar: Proje-spesifik kalıcı bilgiyi (mimari karar, bilinen bug, credential, çalışan komut) kaybetmeden saklar ve BM25 ile aranabilir kılar.

Plugin'deki rolü

Plugin'in kalbi. 5 skill (init/note/search/recall/audit) ve 3 agent (writer/initializer/searcher) doğrudan Obsidian üzerine kurulu; 4 hook (context/daily/userprompt-hint/stop-daily) vault ile konuşur. Vault ~/Documents/ObsidianVault/<proje>/ altında MOC + wikilink yapısındadır. CLI kapalıyken filesystem fallback'e (Read/Grep) düşer.

Plane
plane-cli · proje/issue yönetimi

Nedir: Açık kaynak, self-hosted bir proje/issue yönetim aracı (Jira/Linear alternatifi). Plugin plane-cli binary'si üzerinden REST API'sine bağlanır.

Ne işe yarar: Issue/project/cycle/module listeleme, oluşturma, state güncelleme, atama ve yorum — terminalden çıkmadan iş takibi.

Plugin'deki rolü

Opsiyonel — CLAUDE.local.md'de Plane proje UUID tanımlıysa: init.md kuralı sorun bildirilince ilgili issue'yu arar; commit skill'i değişiklikle uyuşan issue'yu completed state'e çekebilir. Tanımlı değilse Claude native TaskCreate ile takip yeterli.

Yapılandırma
~/.zshrc → PLANE_URL + PLANE_API_KEY + PLANE_WORKSPACE_SLUG
CLAUDE.local.md → Plane proje UUID (opsiyonel)
Solo
CLI · process yönetimi

Nedir: Geliştirme process'lerini (backend, frontend, worker vb.) yöneten bir araç; solo.yml ile tanımlanan komutları başlatır, durdurur, yeniden başlatır ve çıktılarını takip eder. solo CLI binary'si Solo app'in HTTP control plane'ine bağlanır (solo doctor ile doğrulanır).

Ne işe yarar: Tekrar eden shell komutlarını (build + run) tek yerde toplayıp Claude'un bunları başlatıp loglarını okumasını sağlar.

Plugin'deki rolü

soloterm.md kuralı: tekrar eden komutlar için mutlaka solo.yml oluşturulur ve komutlar Solo üzerinden çalıştırılır. Process başlatma/durdurma/restart solo processes / solo commands CLI komutlarıyla yapılır; process çıktısı/logu artık solo processes output ile CLI'den okunur (v0.8+; --lines/--raw destekli) — eski GUI/log-dosyası workaround'ı ve geçici Solo MCP yara bandı kaldırıldı. (Bu plugin repo'sunda tek process var: render — PA Render server.)

PA Render
Bun server · canlı analiz dashboard'u

Nedir: scripts/render-server.ts — Bun ile yazılmış sıfır-bağımlılık HTTP server (127.0.0.1:4787). ~/.pa-render/active|archive/<konu>/index.html sayfalarını tek ekranlı dashboard'da sunar: sol sidebar (aktif + katlanabilir arşiv listesi, görülmemiş-değişiklik dot'u), sağ iframe, SSE ile auto-reload — agent dosyayı Edit'leyince ekran kendini tazeler.

UI kit: render/lib/ — pa.css design system (çift tema, B-section kartları, badge/callout/tablo/oq/risk) + pa.js web component'leri (pa-flow, pa-step, pa-leg, pa-compare, pa-kpi, pa-timeline) + vendored Chart.js ve preact/htm. Sayfalar /lib/*'ı link'ler — inline CSS yok. .jsx dosyaları istekte anında transpile edilir.

Plugin'deki rolü

user-render skill'inin sunum katmanı. Solo process render ile başlatılır (solo.yml); arşivle/geri-al hem UI butonlarıyla hem /api/archive endpoint'iyle yapılır. Agent server'ı hiç bilmez — yalnız dosya yazar.

WhatsApp
whatsapp-mcp · mesaj kanalı

Nedir: WhatsApp Web bridge'i üzerinden mesaj gönderme/okuma sağlayan bir MCP server (whatsapp-mcp, stdio). QR kod okutulup bridge başlatılarak bağlanır.

Ne işe yarar: Sohbetleri listeleme, mesaj arama, kişi bulma ve mesaj/dosya/ses gönderme — Claude'dan doğrudan WhatsApp iletişimi.

Plugin'deki rolü

user-message skill'inin çıktı kanallarından biri. Skill müşteri/acenta mesajını kanal-agnostik üretir; WhatsApp seçilirse mcp__whatsapp__* araçlarıyla gönderilir (skill mesajı üretir, gönderimi entegrasyon yapar).

Google Workspace
Gmail · Calendar · Drive (claude.ai MCP)

Nedir: claude.ai üzerinden sağlanan resmi Google MCP server'ları — Gmail (e-posta), Google Calendar (takvim) ve Google Drive (dosya). HTTP MCP olarak bağlanır, OAuth ile yetkilendirilir.

Ne işe yarar: E-postaları okuma/arama/gönderme, takvim etkinliklerini görüntüleme/oluşturma ve Drive dosyalarına erişim — Claude'dan doğrudan Google hesabıyla çalışma.

Plugin'deki rolü

user-message skill'inin e-posta kanalı Gmail ile gönderilebilir; takvim/dosya entegrasyonu genel asistan akışlarını besler. Üç servis de claude.ai üzerinden gelir — ayrı binary kurulumu gerekmez.

Yetkilendirme
Claude Code → /mcp → claude.ai servisi seç → Authenticate
# Gmail / Google Calendar / Google Drive ayrı ayrı OAuth
Diğer Araçlar
opsiyonel destekleyiciler

Plugin ayrıca şu araçlarla entegre olur (hepsi opsiyonel, setup.sh kurar):

Destekleyici stack
  • uv — Python paket/script yöneticisi; setup.py, crawl2md ve tüm Python akışlarının temeli (pip yasak, kural gereği).
  • Playwright CLI — tarayıcı otomasyonu ve frontend testi (browser-testing kuralı; MCP yerine CLI).
  • playwright-best-practices — @playwright/test ile E2E/component/API/visual spec yazımı için harici skill (currents.dev, MIT); ~/.claude/skills/ altına klonlanır.
  • Context7 (ctx7) — kütüphane/framework dökümanını canlı çeker; obsidian-doc-source library modunda zorunlu.
  • Plane (plane-cli) — proje/issue yönetimi; user-message ile issue yanıtı üretimi.
  • markitdown — PDF/Office/ses/görsel → markdown; crawl2md ve obsidian-doc-source dosya kaynaklarında.
